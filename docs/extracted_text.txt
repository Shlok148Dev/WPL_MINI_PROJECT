2. GitHub Workflow — Complete Guide
This section covers everything from first-time setup to daily workflow. Read this before touching any code.
2.1  One-Time Setup (Do This First — All Three of You)
Step 1 — Install Git
Download and install Git from https://git-scm.com/downloads. During installation on Windows, keep all default options. After installing, open a terminal (Command Prompt or VS Code terminal) and verify:
git --version    # should print something like: git version 2.x.x
Step 2 — Configure Your Identity (Each Person Does This Once)
Run these two commands in your terminal, using your own name and the email linked to your GitHub account:
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
Step 3 — Create the GitHub Repository (Shlok Does This)
Go to github.com and sign in
Click the green 'New' button (top left)
Repository name: jewelry-store
Set visibility to Private (add Spriha and Arnav as collaborators after)
Check 'Add a README file'
Click 'Create repository'
Step 4 — Add Collaborators (Shlok Does This)
Open the repository on GitHub
Go to Settings > Collaborators > Add people
Search for Spriha's and Arnav's GitHub usernames and add both
They will receive an email — they must accept the invite before they can push code
Step 5 — Clone the Repository (All Three Do This)
Everyone clones the repo to their own computer. On the GitHub repo page, click the green 'Code' button and copy the HTTPS URL, then run:
git clone https://github.com/YOUR-USERNAME/jewelry-store.git
cd jewelry-store
This creates a jewelry-store/ folder on your computer that is connected to GitHub.

2.3  Daily Workflow — What to Do Every Time You Work
Follow these steps every single time you sit down to code. This keeps everything in sync.
Before You Start Coding — Pull Latest Changes
git checkout main          # switch to main branch
git pull origin main       # get latest updates from GitHub
git checkout your-branch   # switch back to your branch  (e.g. shlok / spriha / arnav)
git merge main             # bring any new changes into your branch
While Coding — Save Your Work to GitHub
After completing a meaningful chunk of work (a feature, a page, a fix), run:
git add .                          # stage all your changes
git commit -m 'Add search page'   # describe what you did (keep it short & clear)
git push origin your-branch       # push to GitHub
Good Commit Message Examples
Add database schema

At the End of Your Day — Push Everything
git add .
git commit -m "End of day — work in progress on contact page"
git push origin your-branch
2.4  Merging Into Main (Do This on Integration Day — Day 5)
When a team member's work is fully tested and ready, merge it into main so everyone gets the latest combined code.
How to Merge (Example: Shlok Merging His Branch)
git checkout main          # switch to main
git pull origin main       # make sure main is up to date
git merge shlok            # merge shlok's branch into main
git push origin main       # push the updated main to GitHub
Repeat this for each person's branch: spriha and arnav.
Recommended Merge Order on Day 5

Spriha merges second — CSS is needed by everyone's pages
Arnav merges last — form pages depend on shared CSS being present
2.5  Handling Merge Conflicts
A conflict happens when two people edit the same part of the same file. This is rare if you each own separate files, but it can happen in shared files like README.md.
Git will mark the conflict inside the file like this:
<<<<<<< HEAD
Your version of the line
=======
Other person's version of the line
>>>>>>> their-branch
To fix: open the file, delete the conflict markers, keep the correct version, then:
git add .
git commit -m "Resolve merge conflict in README"
git push origin main
Best way to avoid conflicts: each person only edits their own files. Only README.md and index.html may need coordination.

2.7  GitHub in VS Code (Easier Alternative to Terminal)
If you prefer not to use the terminal, VS Code has built-in Git support:

Click the 'Source Control' icon on the left sidebar (looks like a branch)
You'll see all changed files listed — click '+' to stage them
Type a commit message in the box and press Ctrl+Enter to commit
Click the '...' menu and select 'Push' to push to GitHub
You can also install the 'GitHub Pull Requests' extension for a more complete experience.

Part 1 — Database Design
Task 1: schema.sql — 4 Normalized Tables
-- 1. CATEGORIES
CREATE TABLE categories (
    category_id   SERIAL PRIMARY KEY,
    category_name VARCHAR(50) NOT NULL,
    description   TEXT
);
-- 2. PRODUCTS
CREATE TABLE products (
    product_id      SERIAL PRIMARY KEY,
    product_name    VARCHAR(100) NOT NULL,
    description     TEXT,
    price           DECIMAL(10,2) NOT NULL,
    category_id     INT REFERENCES categories(category_id),
    image_url       VARCHAR(200),
    stock_quantity  INT DEFAULT 0,
    created_date    DATE DEFAULT CURRENT_DATE
);
-- 3. CONTACT_SUBMISSIONS
CREATE TABLE contact_submissions (
    submission_id   SERIAL PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,
    email           VARCHAR(100) NOT NULL,
    message         TEXT NOT NULL,
    submission_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- 4. USER_REGISTRATIONS
CREATE TABLE user_registrations (
    user_id           SERIAL PRIMARY KEY,
    full_name         VARCHAR(100) NOT NULL,
    email             VARCHAR(100) UNIQUE NOT NULL,
    phone             VARCHAR(15),
    registration_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Task 3: queries.txt — Common SQL Queries
-- All products with their category names
SELECT p.product_name, p.price, c.category_name
FROM products p JOIN categories c ON p.category_id = c.category_id;
-- Search by keyword
SELECT * FROM products WHERE product_name ILIKE '%diamond%';
-- Products by category
SELECT * FROM products WHERE category_id = 1;
-- Count per category
SELECT c.category_name, COUNT(p.product_id) AS product_count
FROM categories c LEFT JOIN products p ON c.category_id = p.category_id
GROUP BY c.category_name;

// search.js
let selectedCategory = 'all';
function performSearch() {
  const term = document.getElementById('search-input').value.toLowerCase();
  const results = productsData.filter(p =>
    (p.name.toLowerCase().includes(term) ||
     p.description.toLowerCase().includes(term)) &&
    (selectedCategory === 'all' || p.category === selectedCategory)
  );
  displayResults(results);
}
// Also trigger on Enter key:
document.getElementById('search-input').addEventListener('keypress', e => {
  if (e.key === 'Enter') performSearch();
});

// Helper functions
function validateEmail(email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}
function validatePhone(phone) {
  return /^[0-9]{10}$/.test(phone);
}
// Pattern for every form submit handler:
function submitContactForm(e) {
  e.preventDefault();
  let valid = true;
  const name = document.getElementById('name').value.trim();
  if (name === '') {
    document.getElementById('name-error').style.display = 'block';
    valid = false;
  } else {
    document.getElementById('name-error').style.display = 'none';
  }
  // ... repeat for email and message fields
  if (valid) {
    document.getElementById('success-message').style.display = 'block';
    document.getElementById('contact-form').reset();
    setTimeout(() => document.getElementById('success-message').style.display = 'none', 3000);
  }
}
